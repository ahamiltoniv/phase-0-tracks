Git Workflow steps

  1. Create a feature branch
  2. Make changes on your feature branch
  3. Add or Stage your changes
  4. Make a commit with a good message to "save" your work
  5. Make your pull request to Github

Mistakes that I shouldn't continue to make

  • NEVER use git push origin master to skip a pull request
  • Remember to create a feature branch instead of working in the master
  • Don't forget to frequently add and commit your work
